# AI SDK 仕様書 (v2.0)

このドキュメントは、AI サービス クライアント SDK の標準インターフェースと動作仕様を定義します。これにより、さまざまな AI バックエンド（Ollama、OpenAI など）の実装が統一され、予測可能な API を持つことを保証し、フロントエンドへの統合と置き換えを容易にすることを目的としています。

## コア設計原則

1.  **非同期とイベント駆動**: すべてのネットワークリクエストは非同期操作です。レスポンスのライフサイクルは、コールバック関数（`onStart`、`onStream`、`onComplete`、`onError`）を通じて処理されます。
2.  **セッションの分離**: 各 `sendRequest` 呼び出しは独立したセッションを作成し、一意の `sessionId` を介して識別および管理されます。
3.  **ストリーミング優先**: SDK 設計は、最高のユーザーエクスペリエンスを提供するために、ストリーミングレスポンスを強制または優先します。
4.  **制御可能性**: 進行中の AI 生成タスクをユーザーが中断できるように、信頼性の高いリクエスト終了メカニズム（`abort` および `abortAllSessions`）を提供する必要があります。
5.  **エラー処理**: エラーは `onError` コールバックまたは例外のスローを通じて処理され、可能な限り詳細なコンテキスト情報を含める必要があります。

## インターフェース仕様

### `constructor(config)`

クライアントインスタンスを初期化します。

-   **パラメーター**:
    -   `config` (`object`): 設定オブジェクト。
        -   `endpoint` (`string`): **必須**。AI サービスの API エンドポイント URL。
        -   `apiKey` (`string`): **任意**。認証用の API キー。一部のサービス（OpenAI など）で必要です。
        -   `defaultModel` (`string`): **任意**。モデルが指定されていない場合に使用されるデフォルトのモデル ID。
        -   `service` (`string`): **任意**。特定のサービスタイプ（`"ZhiPu"`、`"Google"` など）を識別し、特殊な処理を行うために使用されます。

-   **動作**:
    -   `endpoint` が存在することを確認する必要があります。存在しない場合はエラーをスローします。
    -   以降のメソッドで使用するために設定を保存します。

### `async getModels(forceRefresh = false)`

サービスがサポートしているモデルのリストを取得して返します。

-   **パラメーター**:
    -   `forceRefresh` (`boolean`): **任意**。デフォルトは `false`。`true` の場合、キャッシュを無視してサーバーから強制的に再取得します。
-   **戻り値**: `Promise<string[]>` - モデル ID 文字列を含む配列を返します。
-   **動作**:
    -   不要なネットワークリクエストを避けるため、実装ではモデルリストをキャッシュする必要があります。
    -   リクエストが失敗した場合は、詳細なエラーをスローする必要があります。

### `async sendRequest(messages, options = {})`

チャットリクエストを送信し、レスポンスを処理します。

-   **パラメーター**:
    -   `messages` (`object[]`): **必須**。会話履歴を表すオブジェクトの配列。
        -   各オブジェクトの形式: `{ role: 'user' | 'assistant' | 'system', content: '...' }`
    -   `options` (`object`): **任意**。リクエストの詳細な設定が含まれます。
        -   `model` (`string`): このリクエストで使用するモデル ID。指定されない場合は `defaultModel` が使用されます。
        -   `onStart(sessionId)` (`function`): リクエスト開始時に直ちに呼び出され、生成された一意の `sessionId` を返します。
        -   `onStream(chunk, fullText, sessionId)` (`function`): データストリームの各チャンクを受信するたびに呼び出されます。
            -   `chunk`: 現在受信したテキストフラグメント。
            -   `fullText`: これまでに受信した全テキストの結合。
            -   `sessionId`: 現在のセッション ID。
        -   `onComplete(fullResponse, sessionId)` (`function`): ストリームが終了し、リクエストが正常に完了したときに呼び出されます。
        -   `onError(error, sessionId)` (`function`): ネットワークまたは API エラーが発生したときに呼び出されます。
        -   `temperature` (`number`): モデル出力のランダム性を制御します。通常は 0 から 2 の範囲です。
        -   `options` (`object`): バックエンド固有のプロプライエタリなパラメーター（Ollama の `num_ctx` など）を渡すためのコンテナ。

-   **戻り値**: `Promise<string>` - このリクエストの `sessionId` を返します。

### `abort(sessionId)`

進行中のリクエストを終了します。

-   **パラメーター**:
    -   `sessionId` (`string`): **必須**。終了するセッションの ID。
-   **戻り値**: `boolean` - セッションが存在し、終了信号が正常に送信された場合は `true` を返し、それ以外の場合は `false` を返します。

### `abortAllSessions()`

進行中のすべてのリクエストを終了します。

-   **戻り値**: `number` - 正常に終了したセッションの数。

## ベストプラクティス

-   **`AbortController` の実装**: `abort()` および `abortAllSessions()` は、内部で `AbortController` を使用して `fetch` リクエストをキャンセルする必要があります。
-   **セッションのクリーンアップ**: リクエストが完了（`onComplete`）または失敗（`onError`）した後、対応する `sessionId` を内部のセッション管理（`activeSessions` Map など）から削除する必要があります。
-   **パラメーターの検証**: `messages` などの重要な入力に対して基本的な形式検証を実行します。
-   **ロギング**: デバッグを容易にするために、主要なステップ（リクエストの失敗、解析エラーなど）でコンソールログを追加します。
